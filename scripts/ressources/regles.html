                Comment créer une règle
                
            

            

                

                    
1 Nom de la règle

                    
Donnez un nom court et descriptif (ex. Administratif, Formateurs). Ce nom devient le préfixe de tous les fichiers CSV et groupes AD générés.

                

                

                    
2 Règle active / inactive

                    
Une règle inactive ne peut pas générer de CSV. Elle reste visible dans la liste (section séparée, grisée) mais est suspendue. Utilisez ce toggle pour désactiver temporairement sans supprimer.

                

                

                    
3 Niveau de groupement

                    
Détermine comment les utilisateurs sont répartis entre les fichiers CSV et les groupes AD.

                    

                        

                            1
                            Global
                            1 seul fichier CSV avec tous les utilisateurs correspondants.
→ 1 groupe AD global.
                        

                        

                            2
                            Par DO
                            1 CSV par direction opérationnelle + 1 CSV global.
→ 1 groupe AD par DO (membres = utilisateurs) + 1 groupe global (membres = groupes DO).
                        

                        

                            3
                            Par centre
                            
                                Complet : 1 CSV par centre + 1 CSV par DO + 1 CSV global.

                                → Hiérarchie complète : groupe centre (utilisateurs) → groupe DO (groupes centres) → groupe global (groupes DO).


                                Par centre uniquement : 1 CSV par centre seulement (sans DO ni global) — cochez l'option dédiée.
                            
                        

                    

                    
Les directions opérationnelles sont normalisées automatiquement : DO NORD et DO IDF sont regroupées sous DO I2N, DO SUD et DO SUD-EST sous DO SUD, etc.

                

                

                    
4 Conditions d'inclusion

                    
Définissez les critères que les utilisateurs doivent satisfaire pour être inclus. Si plusieurs conditions sont ajoutées, elles fonctionnent en OU : un utilisateur est inclus s'il correspond à au moins une condition.

                

                

                    
5 Conditions d'exclusion

                    
Parmi les utilisateurs inclus, retirez ceux qui correspondent à l'une des conditions d'exclusion. Comme pour l'inclusion, les exclusions fonctionnent en OU.

                

                

                    
6 Opérateurs disponibles

                    

                        
est exactement — correspondance exacte (insensible à la casse)

                        
n'est pas — différent de la valeur

                        
contient — utilise un joker *valeur*

                        
ne contient pas — exclut si la valeur est contenue

                    

                

                

                    
7 Sélecteur de valeurs

                    
En cliquant sur le champ valeur, une liste déroulante affiche les valeurs réelles présentes dans le cache AD. Vous pouvez :

                    
    
                        
  • Saisir du texte pour filtrer la liste
    • 
                        
  • Cliquer directement sur une valeur pour la sélectionner en un clic
    • 
                        
  • Cocher plusieurs valeurs pour créer plusieurs conditions d'un coup
    • 
                        
  • Cliquer sur Valider pour insérer toutes les valeurs cochées comme autant de lignes
    • 
                    

                

                

                    
8 Générer les CSV

                    
Une fois la règle enregistrée, cliquez sur Générer le CSV. L'AD est interrogé en lecture seule, les utilisateurs sont filtrés selon les conditions, puis les fichiers sont écrits dans application/output/ dans un sous-dossier horodaté.

                    
    
                        
  • Chaque fichier contient : Nom, SAM Account Name, Mail, Fonction.
    • 
                        
  • À la fin, une modale liste tous les fichiers classés par hiérarchie (global → DO → centre).
    • 
                        
  • Cliquer sur un fichier affiche son contenu (Nom + Fonction des utilisateurs).
    • 
                    

                

                

                    
9 Simulation provisionnement groupes AD

                    
Après chaque génération CSV, la console affiche une simulation complète du provisionnement des groupes de distribution AD (tag [AD-PROV]). Les opérations AD sont commentées — aucune modification n'est effectuée.

                    
    
                        
  • Étape 1 : liste de tous les groupes à créer (New-ADGroup) avec leur mail et leur type (parent / feuille).
    • 
                        
  • Étape 2 : alimentation de chaque groupe — les groupes feuilles (centre) reçoivent les utilisateurs du CSV (Add-ADGroupMember) ; les groupes parents (DO, global) reçoivent leurs sous-groupes comme membres.
    • 
                    

                    
Exemple pour la règle FORMATEURS au niveau 3 :

                    
FORMATEURS              ← groupe global (membres = groupes DO)
  FORMATEURS-DO-I2N     ← groupe DO    (membres = groupes centres)
    FORMATEURS-DO-I2N-CENTRE-LILLE  ← groupe centre (membres = utilisateurs)
    FORMATEURS-DO-I2N-CENTRE-PARIS
  FORMATEURS-DO-OUEST
    FORMATEURS-DO-OUEST-CENTRE-NANTES
