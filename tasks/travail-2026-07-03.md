## 1. Nommage des fichiers CSV

- Chaque fichier CSV est nommé d'après **l'adresse mail complète** du groupe (avec le
  domaine), pour chaque niveau : global / DO / centre.
  - ex. `personnes-adm@aftral.com.csv`, `personnes-adm-est@aftral.com.csv`,
    `personnes-adm.appoigny@aftral.com.csv`
- Implémenté dans `Write-RuleCsvSet` (helper `$mailBase = $Id.mail`) → s'applique
  partout (génération par paire **et** « Générer tous les CSV »).

## 2. Nom des dossiers de génération

- Dossier horodaté **sans** suffixe `_TOUS-LES-GROUPES` et **sans les secondes** :
  `2026-07-03_15-32` (format `yyyy-MM-dd_HH-mm`).

## 3. Fonctionnalité DELTA CSVs

**Bouton** : « Générer tous les CSVs avec DELTA » → ouvre une **modale** :
- **Colonne gauche (33 %)** : liste des dossiers existants avec un **radio** ; le plus
  récent est pré-coché (sélectionnable).
- **Zone droite** : notice explicative + case **« Comparer avec le dossier sélectionné »**
  (cochée par défaut). Modale agrandie + colorée (en-tête dégradé bleu).

**Traitement au clic « Générer »** :
1. Génère tous les CSV dans un nouveau dossier horodaté (historique) — barre de progression bloquante.
2. Si comparaison cochée → **delta** entre la génération courante et le dossier de référence.

**Delta** (`Write-CsvDelta` dans `csv-generator.psm1`) :
- **Clé de comparaison = adresse mail**.
- Résultat écrit dans `<run>\__DELTA CSVs -- <date-du-dossier-référence>\`, **même
  arborescence** `GROUPE\<mail>.csv`.
- Chaque fichier delta : en-tête `samAccountName;mail;type`, lignes `…;ajout` ou `…;suppression`.
- Seuls les fichiers présentant au moins une différence sont écrits.

**Convention (validée par test contrôlé)** :
- **ajout** = présent dans la génération courante, absent de la référence → à **ajouter** dans l'AD
- **suppression** = présent dans la référence, plus dans la génération → à **retirer** de l'AD
- Règle d'or : **référence = état déjà en place**, génération = nouvel état souhaité.

**Endpoints backend** : `POST /api/csv/delta` (sécurisé dans `application\output`).

## 4. Ergonomie de l'onglet Fichiers CSVs

- **Arborescence** de la zone du bas : regroupement par **GROUPE** (en-têtes repliables,
  comportement **accordéon** — un seul groupe ouvert à la fois), compteur par groupe.
- **`__DELTA CSVs`** : trié **en tête**, **sticky**, style violet, police plus grande.
- Fichiers d'un groupe ouvert : DIV scrollable (~10 lignes), **fond gris clair**,
  `margin-right: 15px`.
- Groupe **ouvert** surligné en couleur.
- **Bandeau** au-dessus du champ de recherche = **nom du dossier sélectionné** (taille 15px).
- En-tête de la zone de résultats (droite) : **nom du fichier CSV** à gauche, nb de lignes à droite.
- **Badge Δ** sur les dossiers contenant un delta.
- Zone de gauche **redimensionnable** (poignée, largeur mémorisée).
- **Suppression** d'un dossier (corbeille + confirmation) → `POST /api/output/delete`.
- **Ouvrir dans l'explorateur Windows** (icône dossier) → `POST /api/output/open`.

## 5. Compteurs de la sidebar Règles = instantanés

- La sidebar Règles lit désormais les compteurs **mémorisés** dans le cache HTML des
  groupes (`meta` / fichiers `.sig`) → **instantané**, au lieu du lent `/api/regles/counts`
  (~26 s), qui ne sert plus que de repli si un compteur manque.

## 6. Garde-fou : génération sur cache vide

- Nouvel endpoint `GET /api/cache/ready` (nombre d'utilisateurs en cache).
- Avant toute génération CSV, refus si le cache est **vide/absent** (sinon groupes vides
  → delta géant de « suppressions »). Message clair invitant à reconstruire le cache.

## 7. Blocage de l'interface pendant la reconstruction du cache (Explorateur AD)

- Le **↻ Cache de l'Explorateur AD** (`refreshAllCache`) bloque désormais **toute
  l'application** via le bandeau du shell (même overlay que la génération GROUPES).
- Déroulé : bandeau bloquant + barre de progression (site en cours) → **attente réelle**
  du prefetch des 172 sites → reconstruction **synchrone** du cache global
  (`/api/users/preload`) → déblocage → `cache-rebuilt`.
- **Corrige le footer « Cache : absent »** : le cache global `_users_global.json` est
  reconstruit **avant** le déblocage, donc le footer affiche la **date** et non « absent ».
- L'overlay du shell supporte un mode **indéterminé** + un champ `status`.
- L'onglet **Règles** n'a **pas** été modifié (rollback effectué).
