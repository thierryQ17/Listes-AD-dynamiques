## 1. Fix — modale « groupes d'une DO » : chevauchement des membres

Symptôme (mode DDG-only) : dans la modale, les noms des membres **se chevauchaient** sur
2 colonnes, puis un **ascenseur horizontal** parasite apparaissait (noms coupés à gauche).

- **Cause 1 (chevauchement)** : les cartes **clonées** dans la modale ne sont pas dans
  `.do-centres` → elles héritaient de `.members{columns:2}` au lieu du `columns:1` du corps
  principal. → forcé `columns:1` dans `.do-modal-body`.
- **Cause 2 (barre horizontale)** : poser `overflow-y:auto` **seul** fait **promouvoir
  `overflow-x` à `auto`** (règle CSS : un axe non-`visible` fait passer l'autre `visible` à
  `auto`). → `overflow-x:hidden` explicite + colonne nom **flexible** (`minmax(0,1fr) auto`)
  pour tronquer (…) au lieu de déborder.
- Résultat : **1 colonne**, **max ~10 lignes** (`max-height:178px`), **scroll vertical seul**.

Fichier : `scripts/ressources/groups-doc.js`. Vérifié `node --check`.

## 2. Investigation — exécuter `/clear` automatiquement (prouvé impossible)

Demande : qu'une commande écrive le récap **puis lance `/clear` toute seule**, voire
**pré-remplisse** la ligne de saisie.

**Vérifié** (sous-agent `claude-code-guide` + doc officielle Claude Code), pas supposé :

| Voie | Possible ? |
|---|---|
| Skill déclenchant `/clear` | ❌ un skill = instructions, pas un exécuteur de slash-commands |
| Hook (`Stop`, `SessionEnd`…) déclenchant `/clear` | ❌ aucun champ de sortie de hook ne vide le contexte |
| Flag CLI / réglage `settings.json` | ❌ inexistant |
| **Pré-remplir la zone de saisie** | ❌ l'assistant n'a aucun accès à l'input (ni skill, ni hook) |
| `/clear` manuel en session | ✅ saisi par l'utilisateur uniquement |
| SDK Agent : `query(prompt:"/clear", continue:true)` | ✅ mais **script externe**, pas en TUI |

**Conclusion** : en terminal interactif, le `/clear` **reste une frappe manuelle** ; seul le
récap est automatisable. (C'est aussi plus sûr : on relit le récap avant d'effacer le contexte.)

Skill **global** créé : `~/.claude/skills/fin-session/SKILL.md` (**hors dépôt** — non versionné
ici). Workflow :

1. Horodatage (`date`) → `docs/RECAP-aaaa-mm-jj_hh-mm.md`.
2. Rassemble la matière (commits depuis le dernier RECAP + fil de conversation).
3. Rédige le récap au **gabarit maison** (intro lecture seule, sections thématiques avec

4. **Commit + push** (`docs: récap de session <horodatage>`, Co-Authored-By).

Format du nom de fichier étendu avec l'**heure** (`_hh-mm`) → plusieurs récaps possibles le
même jour.
