Objectif : symétrique de `session-fin` (début ↔ fin de session).

**Débat preuve / contre-preuve** mené avant de coder :

  hook `SessionStart(clear)` **injecte déjà** ce fichier après un `/clear` (script générique

RECAP**, **confrontation à l'état réel du dépôt** (`git status` / `log <base>..HEAD` / `diff
--shortstat`), **briefing court** (où on en est / points ouverts / prochaine action).

Ajustement demandé ensuite : **retrait** de l'étape « Vigilance » (redémarrage Pode / rappel
lecture seule) ; frontmatter `description` nettoyé en cohérence.

## 3. Globalisation du hook de reprise après `/clear`

**Question** : gérer `CONTEXTE-SESSION.md` + `docs/RECAP-*.md` dans **tous** les projets via un
**hook global** ?

**Analyse (preuve / contre-preuve)** — un hook ne « gère » qu'une **moitié** :
- **Générer** les documents = rôle de la skill `session-fin` (le *modèle* rédige) → **déjà
  global**. Un hook, code déterministe, **ne peut pas** rédiger un récap.
- **Réinjecter** `CONTEXTE-SESSION.md` après `/clear` = rôle du hook → c'était le seul manque
  hors de ce projet.
- **Vérifié** : le `settings.json` global a déjà un `SessionStart` (crée `tasks/`) → mécanisme
  global prouvé. Et `resume-context.ps1` est **déjà générique** (`$env:CLAUDE_PROJECT_DIR`,
  `Test-Path`) → réutilisable tel quel, inoffensif là où le fichier est absent.

**Actions réalisées** (option recommandée, validée) :
- Script copié en **`~/.claude/hooks/resume-context.ps1`** (en-tête génériques « tous projets »).
- **Hook global** ajouté dans **`~/.claude/settings.json`** : `SessionStart` / `matcher "clear"`
  → `pwsh -NoProfile -File C:/Users/thgadre/.claude/hooks/resume-context.ps1` (chemin **absolu**,
  car un chemin relatif `.claude/...` se résoudrait depuis le projet courant).
- **Hook projet retiré** : `.claude/settings.json` réduit à `{}` **et** script projet
  `.claude/hooks/resume-context.ps1` supprimé (orphelin) → **plus de double déclenchement**.
