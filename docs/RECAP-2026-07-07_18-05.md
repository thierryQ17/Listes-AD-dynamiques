## 1. Reprise automatique du contexte après `/clear`

Besoin : après un `/clear`, repartir d'une session vierge **mais** avec le contexte du projet.

- **Mécanisme retenu (vérifié, pas supposé)** : un **hook `SessionStart`** avec matcher
  **`clear`**. Vérifié via sous-agent `claude-code-guide` + doc officielle :
  - `SessionStart` **fire bien sur `/clear`** (`source` = `startup | resume | clear | compact`) ;
  - son **stdout est injecté comme contexte** avant le 1er prompt ;
  - configurable en **`.claude/settings.json` projet** (scopé au projet), commande lancée
    dans le **cwd** (racine projet).
- **`.claude/hooks/resume-context.ps1`** : lit un fichier de contexte et l'émet sur stdout.
- **`.claude/settings.json`** : `SessionStart` / `matcher: "clear"` → lance le script via `pwsh`.

### Deux bugs corrigés au test (heureusement testé)
- **Mauvais fichier choisi** : le tri `Sort-Object Name` de PowerShell est **culture-aware**
  (`.` vs `_` mal ordonnés) → il prenait `RECAP-2026-07-07.md` au lieu du `_17-49`.
  Corrigé en triant par **`LastWriteTime`** (le plus récemment écrit).
- **Accents manglifiés** (`R�capitulatif`) : sortie non-UTF-8 → forcé
  `[Console]::OutputEncoding = UTF8` + `Get-Content -Encoding UTF8`.

## 2. `/clear` automatique : **prouvé impossible** en TUI

Demande insistante d'automatiser le `/clear` (voire de **pré-remplir** la saisie). Vérifié
auprès de la doc (2 passes de sous-agent) :

| Voie | Possible ? |
|---|---|
| Skill / hook déclenchant `/clear` | ❌ un skill = instructions ; aucun hook ne vide le contexte |
| Pré-remplir la ligne de saisie | ❌ l'assistant n'a aucun accès à l'input |
| SDK Agent `query(prompt:"/clear", continue:true)` | ✅ mais **script externe**, pas en TUI |

→ Le `/clear` **reste une frappe manuelle** ; seul le récap + la reprise sont automatisés.
(Plus sûr : on relit le récap avant d'effacer.)

## 3. Design final — « Option 1 » : réutiliser `CONTEXTE-SESSION.md`

Question posée : fusionner tous les recaps en un `RECAP-global.md` ?

**Découverte (`/analyse`)** : le dépôt a **déjà** `CONTEXTE-SESSION.md` (≈ 549 lignes) qui EST
la vue d'ensemble durable (archi, routes, fonctionnalités, conventions, pièges). Un
`RECAP-global.md` cumulé ferait **doublon**, et une **concaténation non bornée** réinjectée à
chaque `/clear` **annulerait le bénéfice** du clear (contexte qui explose).

**Retenu (Option 1)** — séparer les rôles :
- **Recaps datés** = journal chronologique (traçabilité).
- **`CONTEXTE-SESSION.md`** = vue d'ensemble **bornée et curée** → c'est **elle** que le hook
  réinjecte après `/clear`.
- **`/fin-session`** entretient `CONTEXTE-SESSION.md` (synthèse réécrite) **et** écrit le recap daté.

→ Hook repointé sur `CONTEXTE-SESSION.md` (testé : 549 lignes, accents OK). Skill `fin-session`
(hors dépôt, `~/.claude/skills/`) enrichi de l'étape « maj `CONTEXTE-SESSION.md` ».
